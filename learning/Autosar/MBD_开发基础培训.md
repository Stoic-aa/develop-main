# 一、行业简介

1. 传统的开发例如基于 STM32 的开发，有一些痛点，例如移植性差，通用性不强，调试问题定位比较麻烦，需要程序烧录到板子中进行调试， STM32 中的 由寄存器开发到标准库开发，再到 HAL库开发，也就是基于 STM32Cubemx 进行配置生成HAL库进行开发
2. 汽车电子中开发代码量巨大、且要求严格，代码质量要求严格，但是目前车款的推出速度也在加快，因此目前汽车电子的 应用层采用的是 基于模型的 软件开发，采用 Matlab 软件中的 Simulink 进行建模，然后生成代码进行开发，只用专注玉逻辑流程方面的工作，图像化仿真，一套逻辑可以生成多款 芯片上的代码，复用性比较强，同时也提高了开发效率和 软件质量，调试难度

# 二、MBD 开发流程

## 2.1 职业划分

1. 系统工程师
2. 软件架构师
3. 软件开发工程师（包含MBD）
4. 集成工程师
5. 测试工程师

## 2.2 开发流程

-  核心是基于 Matlab simulink 的模型开发，并生成对应的代码进行调试开发
-  一般的开发流程是
    1. 需求分析 - 确定功能需求和性能指标
    2. 系统建模 - 使用Simulink建立系统模型
    3. 模型验证 - 通过仿真验证模型正确性
    4. 代码生成 - 自动生成目标代码
    5. 集成测试 - 在目标硬件上进行测试
    6. 部署实施 - 应用到实际产品中

- MBD开发流程
    1. 模式开发
    2. 模型标准测试
    3. 模型在环测试和硬件在环测试
    4. 代码生成集成测试
    5. 台架环境测试
